Comment Sardiner ?




  • Sardine est la lib/API principale contenant différents mini langages pour la composition musicale (SPL, Ziffers, Vortex Tidal, etc) + Possibilités I/O + Mécanisme de récursion temporel.
  • À son import Sardine installe proprement l'environnement de jeu avec la configuration utilisateur et mets toutes les bonnes variables et les bonnes références dans le scope de l'utilisateur du REPL.
  • Tuna est un éditeur textuel optionnel pour Sardine écrit en Rust fait pour les nouveaux arrivants mais utilisé uniquement par Eden et Raphaël :3
    • D'autres éditeurs sont supportés: Vim/Neovim, Emacs, etc...


Quelles utilisations pour Sardine ?

  • Miika Alonen avec Ziffers: Développer des DSLs par dessus les fonctionnalités de Sardine.
  • Kate Sicchio: Contrôler des robots en rythme pour des performances de chorégraphie contemporaines à New York.
  • Jack Armitage: Utiliser des bibliothèques de machine learning pour des performances audiovisuelles.
    • Contrôler le piano à résonnateur magnétique d'Andrew Mc Pherson
  • Ralt144MI (aka notre Rémi national): Contrôler du matériel audio obsolète et des synthétiseurs vidéo modulaires.
  • De manière générale: Algorave, improvisation algorythmique en live, faire des bétises avec du code et des sons, et de l'impro centré sur l'humain et pas la machine.

Le code Sardine peut être écrit de deux façons différentes :

  • Surfing Functions / Players : une syntaxe simple et limitée, orientée performance en live.
  • Swimming Functions : ressemble à du Python régulier, verbeux mais puissant. Une passerelle vers la récursion temporelle en Python.

Changer la période (1 period = 1 beat) 
				      
Pa * d('bd cp', p=0.5) # La période est de 0.5, 2 notes par battement
Pa * d('bd cp hh:2', p=0.5) # ajoute le sample hh:2, le temps est maintenant ressenti en 3 battements
Pa * d('bd cp hh:2', p=0.25) # La période est de 0.25, 4 notes par battement, le rythme est doublé
Pa * d('bd!2 cp hh:2 .', p=0.25) # ajoute bd repeat et un rest "." - le temps est maintenant ressenti en 5 battements

Voici les senders disponibles avec Sardine:

| Player | @swim | Purpose and description | |:-------:|:-------:|:----------| | **Sardine** | **Senders** | senders used with Sardine patterns | | d( ) | D( ) | **SuperDirt** - samples and synthesizers | | n( ) | N( ) | **MIDI** notes | | cc( ) | CC( ) | **MIDI** control messages | | pc( ) | PC( ) | **MIDI** program changes | | sy( ) | SY( ) | **MIDI** sysex message | | **Ziffers** | **Senders** | senders used with Ziffers patterns | | zd( ) | ZD() | **Ziffers SuperDirt** - samples /synths with Ziffers patterns| | zn( ) | ZN() | **Ziffers MIDI** notes with Ziffers patterns |
Rand, range, ramp, repeat, extend repeat, etc... 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd . sd", i=i)
					    D("synth",
					        midinote="C4|E4 C4~G4 C4!3 [D4 F4 A4]!!4",
					        cutoff="[100:1000, 100]",
					        i=i)
					    again(pat, p=1, i=i+1)
Le pipe permet de lancer une pièce à pile ou face pour choisir entre C et E.
Le tilde crée une rampe entre deux valeurs extrêmes (ici, un chromatique ascension de C à G).
Le point d'exclamation indique de répéter la valeur sur laquelle il est appliqué un certain nombre de fois.
La virgule et les crochets permettent de générer une plage de valeurs équiréparties, ici des valeurs de coupure pour un filtre passe-bas allant de 100 à 1000 par pas de 100 (100, 200, 300 etc).
