Démo

  • Sardine Core is the main library/API containing small languages for musical improvisation (SPL, Ziffers, Vortex Tidal, etc) + I/O features + temporal recursion mechanism.
  • Sardine is a customised version (asynchronous) of the CPython interpreter. It will import Sardine automatically with user config.
  • Sardine Web is an optional text editor for Sardine written in TypeScript made for newcomers but used only by me :)
    • Other editors are supported: VSCode Plugin, Vim/Neovim, Emacs, etc...


What is Sardine used for?

  • Claude: controlling GLSL shaders and syncing them to music.
  • Ziffers: developing DSLs on the top of Sardine core features.
  • Kate Sicchio: Controlling robots in rhythm for contemporary choregraphy performances in NYC.
  • Jack Armitage: using machine learning libraries for audiovisual performances.
    • Controlling Andrew Mc Pherson's magnetic resonator piano.
  • HighHarmonics: electroacoustic music using Sardine + Max/MSP.
  • Ralt144MI: controlling obsolete audio hardware and modular video synthesizers.
  • Broadly speaking: Algorave, live algorithmic improvisation, noodling around with code, sound, and human centric improv.

Sardine code can be written in two different ways:

  • Surfboards / Players : a simple, limited syntax, live oriented.
  • Swimming Functions : looks like regular Python, verbose but powerful. A gateway to temporal recursion in Python.

Changing the period (1 period = 1 beat) 
				      
Pa * d('bd cp', p=0.5) # sets period to 0.5, yields 2 notes per beat
Pa * d('bd cp hh:2', p=0.5) # adds the hh:2 sample, meter is now felt in 3
Pa * d('bd cp hh:2', p=0.25) # period is 0.25, 4 notes per beat, rhythm is twice as fast
Pa * d('bd!2 cp hh:2 .', p=0.25) # adds bd repeat and a rest "." - meter is felt in 5.

Some of the basic handlers you can use with Sardine:

| Player | @swim | Purpose and description | |:-------:|:-------:|:----------| | **Sardine** | **Senders** | senders used with Sardine patterns | | d( ) | D( ) | **SuperDirt** - samples and synthesizers | | n( ) | N( ) | **MIDI** notes | | cc( ) | CC( ) | **MIDI** control messages | | pc( ) | PC( ) | **MIDI** program changes | | sy( ) | SY( ) | **MIDI** sysex message | | **Ziffers** | **Senders** | senders used with Ziffers patterns | | zd( ) | ZD() | **Ziffers SuperDirt** - samples /synths with Ziffers patterns| | zn( ) | ZN() | **Ziffers MIDI** notes with Ziffers patterns |
Rand, range, ramp, repeat, extend repeat, etc... 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd . sd", i=i)
					    D("synth",
					        midinote="C4|E4 C4~G4 C4!3 [D4 F4 A4]!!4",
					        cutoff="[100:1000, 100]",
					        i=i)
					    again(pat, p=1, i=i+1)
The pipe operator allows to flip a coin to chose between C and E.
The tilde operator creates a ramp between two extremal values (here it results in a chromatic ascent from C to G).
The bang/exclamation point operator instructs to repeat the value it is applied to a certain number of times.
The colon and square brackets allow to generate a range of evenly-spaced values, here cutoff values for a low-pass filter going from 100 to 1000 by steps of 100 (100, 200, 300 etc).

r = incrementation rate 
				      
				      Pa * d('amencutup:[0:20]', p=0.25, leg=.5, r=0.25)
d = divisor, skips events when i % d == 0 
				      
				      Pa * d('amencutup:[0:20]', p=0.25, leg=.5, r=0.25, d=3)
i = increment step 
				      
				      Pa * d('amencutup:[0:20]', p=0.25, leg=.5, r=0.25, d=3, i=2)
l = incrementation limit 
				      
				      Pa * d('amencutup:[0:20]', p=0.25, l=2, leg=.5, r=0.25)
until = repeat n times and stop! 
				      
				      Pa * d('amencutup:[0:20]', p=0.25, leg=.5, until=8)
